Append only. Each entry: date, status (`accepted` / `superseded`), the choice, alternatives, rationale.

## 2026-05-12 — Embedding model: `voyage-3-large`
**Status:** accepted

Voyage `voyage-3-large` ($0.18 / 1M tokens) over `voyage-3` ($0.06).

**Alternatives considered:** voyage-3 (cheaper, decent for English glosses but weaker on short technical terms), OpenAI `text-embedding-3-small` (cheapest, second SDK to manage), local `bge-m3` via Ollama (free, slow, GPU-on-VPS prerequisite).

**Rationale:** Backfill cost is ~$3.30 one-time (trivial). Query embedding cost is ~$0.0000006 per query (negligible). Quality margin on short technical glosses matters more than the 3× cost delta. Single vendor for embeddings + chat reduces operational surface.

---

## 2026-05-12 — Pydantic-first contract, OpenAPI generated
**Status:** accepted

`core/schemas/*.py` is the single source of truth. FastAPI and FastMCP both import from there. `data-model/glintstone-schema-api.yaml` is generated by a script and committed (so external consumers can `git pull` the spec).

**Alternatives considered:** OpenAPI-first with code generation; hand-maintained parallel definitions.

**Rationale:** Python is the project's language; Pydantic is idiomatic; FastAPI auto-derives from Pydantic; FastMCP introspects Pydantic for tool schemas. OpenAPI-first adds a code-generation step. Hand-maintained drifts.

---

## 2026-05-12 — `ToolResponse` envelope, locked shape
**Status:** accepted

Every agent tool returns `ToolResponse[T]` with `summary`, `data`, `sources`, `required_attributions`, `follow_ups`, `interaction_id`, `render_hint`, `generated_at`. `sources` is never empty (computed `SourceRef` as fallback).

**Alternatives considered:** per-tool ad-hoc shapes; OpenAI's tool-result style (plain strings); GraphQL-style nested-object responses.

**Rationale:** Claude Desktop / Claude.ai apps render structured payloads when the shape is consistent. One envelope means one renderer, one validator, one contract test.

---

## 2026-05-12 — Own-rolled citations, CSL/BibTeX deferred
**Status:** accepted

Citations stored as decomposed `Citation` objects (scholar_id, year, publication_short, page_ref, …). Synthesis text uses `[n]` markers. No Anthropic Citations API dependency.

**Alternatives considered:** Anthropic Citations API (ties to one model family); pre-rendered citation strings (lossy, no format-pluggable rendering).

**Rationale:** Future academic-style rendering (Chicago, AAA, BibTeX, CSL-JSON) must be a pure projection of the same data. Decomposed components enable that without storage rework.

---

## 2026-05-12 — Best-guess gate: `completeness ≤ 2 AND ≥3 similar tablets`
**Status:** accepted

`summarize_artifact` only generates a "best guess" section when both conditions hold. Otherwise the structured facts speak for themselves.

**Alternatives considered:** Always include best-guess (more cost, more hallucination risk); never include best-guess (less useful for unlemmatized tablets).

**Rationale:** The most-useful case is exactly when the pipeline is sparse AND we have priors. Outside that window, best-guess content adds noise.

---

## 2026-05-12 — Lazy-persist agent outputs, never pre-bake
**Status:** accepted

`agent_outputs` is populated on first request and reused until superseded. Never run a batch job to pre-generate summaries for the corpus.

**Alternatives considered:** Nightly pre-generation for popular tablets; LRU cache with no DB persistence.

**Rationale:** Pre-baking 353k summaries costs ~$1000+ at Sonnet rates; most won't be viewed. Lazy persist costs nothing for unseen tablets. The `superseded_at` invalidation is straightforward. Cache without DB persistence loses the learning signal (what was generated, what cited what).

---

## 2026-05-12 — Default group limit 5, View-All via cursor
**Status:** accepted

`semantic_search` returns 5 results per entity-type group by default. Each group carries `total` + `cursor` + `view_all` FollowUp.

**Alternatives considered:** 10 per group (more dense, harder to scan); 3 per group (PRD #25 minimum, too narrow for context).

**Rationale:** 5 reads well in Claude Desktop renderings, gives enough signal that `total` becomes meaningful as an invitation to expand. Expansion preserves the query embedding for 5 minutes so View-All is cheap.

---

## 2026-05-12 — MCP HTTP: API-key auth, no OAuth
**Status:** accepted

When P4b ships HTTP-transport MCP, auth is `Authorization: Bearer gs_<key>` only. No OAuth flow.

**Alternatives considered:** OAuth (industry standard for public APIs); IP allowlist; no auth.

**Rationale:** Glintstone has no user accounts. Read-mostly scholarly tool. OAuth adds a refresh-token flow with no payoff. API-key auth is what ORACC and other ANE infrastructure use.

---

## 2026-05-12 — Prompt caching mandatory on synthesis
**Status:** accepted

Anthropic `cache_control: {"type": "ephemeral"}` on every synthesis system prompt. The per-artifact fact list goes in `messages`, not `system`.

**Alternatives considered:** No caching (10× more expensive); 1h TTL (2× write premium without enough query density to amortize).

**Rationale:** System prompt is stable across all artifacts in a session. 5-min TTL matches typical tablet-browsing rhythm. Easy >90% cost reduction.

---

## 2026-05-12 — `prompt_version` = filename + frontmatter `revision`
**Status:** accepted

Major versions are new files (`synthesis.v1.md` → `synthesis.v2.md` on contract-breaking change). Minor revisions bump a `revision:` field in the frontmatter; same filename.

**Alternatives considered:** Filename only (every typo fix becomes a new file); frontmatter only (less git-diffable).

**Rationale:** Filename signals contract breaks; frontmatter signals tuning. `agent_outputs.prompt_version` stores the filename stem; reuse logic doesn't break on tuning.

---

## 2026-05-12 — `agent_outputs` invalidation: GIN on `source_run_ids` + 30-day TTL
**Status:** accepted

Indexed array column for the supersession sweep; 30-day max age as safety net.

**Alternatives considered:** Denormalized join table (faster for very large sweeps, more code); trigger on annotation_runs insert (write amplification on a hot table); no TTL (some bugs hide forever).

**Rationale:** Scale is small (~10k summaries/month max). GIN handles it. TTL is belt-and-suspenders for the case where supersession detection has a gap.

---

## 2026-05-12 — Cursor TTL: 5 minutes
**Status:** accepted

`semantic_search` cursors expire after 5 minutes, matching Anthropic prompt cache TTL and Voyage query-embedding cache.

**Alternatives considered:** 1 hour (staler results); no TTL (cache grows unbounded).

**Rationale:** Typical View-All happens within seconds. 5 minutes is generous. Aligning TTLs across systems simplifies eviction reasoning.

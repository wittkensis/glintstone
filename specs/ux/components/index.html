Design Principles

Accessibility First

Every component meets WCAG AA standards. Keyboard navigation, screen reader support, and clear focus states are built in from the start.

Mobile-First

Components are designed for touch interaction first with 44px minimum touch targets, then enhanced for mouse and keyboard.

Immediate Feedback

Every interaction provides immediate, visible feedback. Users never have to wonder if their action was registered.

Transparent Trust

Provenance and authority are always visible. Users can trace how content was created and validated.

Positive Experience

No wrong answers, only data. Components celebrate contributions and reward honest uncertainty.

Direct Manipulation

Prioritize direct interaction (pan, zoom, tap) over indirect controls. Reduce cognitive distance between intention and action.

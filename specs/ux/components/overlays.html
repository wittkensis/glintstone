Overlay Components

Overlay components present content that floats above the main interface. They handle focus management, backdrop interaction, and dismissal patterns while ensuring users are never trapped.

Modal

Modal displays content that requires user attention or decision. Use for confirmation dialogs, forms, important alerts, and content previews.

Interactive Demo

Click buttons to open different modal variants:

Size Variants

Variant Max Width Use Case
sm 400px Simple confirmations
md 560px Forms, detailed content
lg 720px Complex content
fullscreen 100% Mobile, immersive

Keyboard Interaction

Key Action
Tab Cycle through focusable elements
Shift+Tab Cycle backwards
Escape Close modal
Enter Activate focused button
Accessibility Requirements
  • role="dialog" with aria-modal="true"
  • Title linked via aria-labelledby
  • Description linked via aria-describedby
  • Focus trapped within modal while open
  • Escape key closes (unless persistent)
  • Body scroll disabled when open
  • Focus returns to trigger element on close

Code Example

<div class="modal" role="dialog" aria-modal="true" aria-labelledby="modal-title" aria-describedby="modal-description" data-size="md" > <div class="modal__backdrop" data-action="close"></div> <div class="modal__dialog" role="document"> <header class="modal__header"> <h2 id="modal-title" class="modal__title">Confirm Action</h2> <button type="button" class="modal__close" aria-label="Close dialog"> <!-- close icon --> </button> </header> <div id="modal-description" class="modal__content"> <p>Are you sure you want to perform this action?</p> </div> <footer class="modal__footer"> <button type="button" class="button" data-variant="ghost">Cancel</button> <button type="button" class="button" data-variant="primary">Confirm</button> </footer> </div> </div>

Dismiss Triggers

Trigger Behavior
Click outside Closes popover
Press Escape Closes popover
Click close button Closes popover (if present)
Scroll Optional, configurable
Accessibility Requirements
  • aria-expanded on trigger (true/false)
  • aria-haspopup="dialog" on trigger
  • role="dialog" on popover container
  • Focus moves to popover on open (optional)
  • Escape key closes popover

Props/Attributes

Attribute Type Description
data-placement string Preferred position (top, bottom, left, right)
data-offset number Distance from trigger (px)
data-arrow boolean Show pointing arrow
data-dismiss-on-scroll boolean Close when page scrolls

Tooltip

Tooltip provides brief supplementary information on hover/focus. Use for icon labels, abbreviated text explanation, and keyboard shortcut hints.

Interactive Demo

Hover over icons or focus them to see tooltips:

Save changes (Cmd+S)
Edit transcription
Delete
Get help

Show/Hide Behavior

Trigger Type Show Delay Hide Trigger
Mouse hover 300ms Mouse leaves element
Keyboard focus 100ms Focus moves away
Touch (long-press) 300ms Touch release
Escape key - Immediately hides
Accessibility Requirements
  • role="tooltip" on tooltip element
  • aria-describedby links trigger to tooltip
  • NOT for essential information (supplementary only)
  • Visible text alternative always present (e.g., visually-hidden span)
  • Maximum 8 words - keep content brief
  • No interactive elements inside tooltips

Code Example

<!-- Custom tooltip with full control --> <button type="button" aria-describedby="tooltip-1" class="tooltip-trigger" > <svg aria-hidden="true"><!-- save icon --></svg> <span class="visually-hidden">Save changes</span> </button> <div id="tooltip-1" role="tooltip" class="tooltip" > Save changes (Cmd+S) </div>

Toast

Toast shows transient notifications that auto-dismiss. Use for success confirmations, error notifications, system status updates, and non-critical alerts.

Interactive Demo

Click buttons to trigger different toast types:

Toast Types

New tablet added to your collection.

Your changes have been saved.

Session will expire in 5 minutes.

Failed to save. Please try again.

Behavior

Property Value
Position (desktop) Bottom-right of viewport
Position (mobile) Bottom-center of viewport
Stacking Vertical, newest at bottom
Max visible 3 toasts at once
Auto-dismiss 4-6 seconds (longer for errors)
Pause on hover/focus Yes, resumes on leave
Accessibility Requirements
  • Container uses aria-live="polite" for announcements
  • Content is announced when toast appears
  • Dismiss button is keyboard accessible
  • Does not steal focus from current task
  • Essential information must have alternative display method
  • Error toasts should persist longer or be persistent

Code Example

<!-- Toast container (add to page once) --> <div class="toast-container" aria-live="polite" aria-atomic="true" > <!-- Toasts inserted here dynamically --> </div> <!-- Individual toast --> <div class="toast" data-type="success" role="status"> <svg class="toast__icon" aria-hidden="true"> <!-- success icon --> </svg> <p class="toast__message"> Your changes have been saved. </p> <button type="button" class="toast__dismiss" aria-label="Dismiss" > <svg aria-hidden="true"><!-- close icon --></svg> </button> </div>

Accessibility Requirements
  • <aside> landmark when inline (visible)
  • role="dialog" when overlay/modal variant
  • Focus managed when opening overlay variants
  • Escape key closes overlay variants
  • Content updates announced to screen readers
  • Descriptive aria-label on panel

Task Components

TaskCard

The container component for all task types. Provides consistent framing, progress indication, and action controls for the Contribution-Reward Cycle.

Live Example - Complete a task

Task 3 of 10 0:45

Which sign matches the highlighted area?

Task States

StateDescription
loadingTask content loading
readyAwaiting user input
submittingProcessing response
feedbackShowing result feedback
transitioningAnimating to next task

Footer Actions

ActionBehavior
"I'm not sure"Marks task with uncertainty flag, counts toward progress, shows positive feedback
"Skip"Silent skip, task returns to queue, does NOT count toward progress
Accessibility: Progress bar has proper ARIA attributes. Task prompt linked to content via aria-describedby. Focus managed on task load.

BinaryTask

Presents a yes/no or true/false decision. Use for AI suggestion validation, damage identification, and simple verification tasks.

Variants

VariantLabelsUse Case

Accessibility: Two large buttons with clear labels. Keyboard: Tab between, Space/Enter to select. Minimum button size: 120x60px.

CountTask

Asks the user to count something visible on the tablet. Use for line counting, sign counting, and column identification.

Live Example - Count the lines

Accessibility: Large number buttons (minimum 60x60px). Clear focus states. Semantic radio group.

Manages the list of pending tasks and provides queue visualization. Use for Early Learner task selection and Expert review queue.

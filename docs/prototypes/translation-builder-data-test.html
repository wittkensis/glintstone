Design Gap The "Show analysis layers" toggle exists in View Controls but the design doesn't specify what "on" looks like. In this prototype, language switches are shown as colored top-borders on tokens (blue = Akkadian, green = Sumerian) and a AKK/SUX flag chip. When toggle is off, these would disappear — but no design spec exists for either state. This is the most impactful gap: without language indicators, bilingual lines (which make up ~18% of the corpus) are visually ambiguous.

Improvement Suggestions — 12 Scholar-Informed Ideas

Based on PLANNING research: personas (Dr. Al-Rashid, Aisha Okonkwo, Marcus Chen), domain reports, and the ATF format spec. Priority ratings reflect likely impact × implementation cost.

01 High Priority
Language separator row — define the "Show analysis layers" behavior
The toggle exists but no design specifies what it does. Proposal: when ON, insert a thin full-width band between token rows whenever the active language changes, with a language label chip. This handles both line-level and mid-line switches consistently. When OFF, the bands hide and tokens lose their language-color top borders.

Critical for all bilingual/multilingual tablets (~18% of corpus: Sumerian+Akkadian, Elamite, Hittite).
Dr. Al-Rashid · Aisha Okonkwo
row 3 — [akk tokens…] ────── %sux — SUMERIAN ──────────── row 4 — [sux tokens…] ────── %akk — AKKADIAN ──────────── row 5 — [akk tokens…]
02 High Priority
Structural line types — ruling, editorial note, blank space
$ beginning broken, $ single ruling, $ blank space, $ rest broken are physical/editorial annotations, not text lines. They need three distinct non-token row variants: (a) Ruling — a visual hr with label; (b) Editorial note — full-width italic text with a distinct background; (c) Missing section — a red-tinted banner spanning the full column width.
All scholars

03 Medium Priority
Translation coverage badge per column header
Scholars need to know at a glance how complete the translation is for each column. Add a small N/M translated chip in the column header (seen in prototyped form above). Color-code: green (>90%), amber (50–90%), red (<50%). This is especially important for tablets with patchy matches due to the line_id = NULL data gap, helping scholars prioritize which columns need editorial attention.
Dr. Al-Rashid · Aisha (dissertation work)
Column 1 259/265 ✓ Column 2 12/18 ● Column 3 0/9 ✗
04 Medium Priority
Uncertain token flag — ? / !? in addition to #
ATF uses # (damaged), ? (uncertain reading), and !? (emendation + uncertain). The current Token component only supports #. Tokens like SI#!? have both damage and uncertainty — two distinct scholarly judgments. Add an amber ? flag chip to the Token Flags row, visually distinct from the red #. Also add ! for emendation (corrected reading by modern editor).
Dr. Al-Rashid (quality control)
┌ flags ┐ # ? — damaged AND uncertain ! — emended (corrected) reading SYL — syllabic (existing)
05 High Priority
Sub-line grouping — visual bracket for 1.a / 1.b notation
Sub-lines (1.a, 1.b, 3.b1) represent variant or parallel readings of the same line number, common in administrative texts. Currently they would render as independent rows with no visual connection. Add: (1) a thin left-side bracket in the row-number gutter connecting sibling sub-lines; (2) slight indentation of sub-line token ribbons; (3) the row number column must accommodate 4-character labels (3.b1).
Assyriologists parsing administrative contracts
13 ┬─ [diri ~ = %a na-ra-mu-um] ├─ 13.a beloved └─ 13.b male person 14 ── [diri ~ = %a asz-ra-tu-um]
06 Medium Priority
Inline composite reference badge in row-number gutter
>>Q000057 047 (composite cross-reference) appears on the same ATF line as the text content. There is no component to display this. Proposal: a small Q###### pill at the bottom of the row-number cell, linking to the composite text view. This is essential for understanding where this exemplar fits within the broader text tradition — a core scholarly task (Dr. Al-Rashid's quality control workflow).
Dr. Al-Rashid · Graduate researchers
07 Medium Priority
Pending data shimmer state — distinguish "no data yet" from "no data exists"
Token rows where token_readings = [] currently render empty/silent. Users cannot distinguish: (a) data not yet processed, (b) data is genuinely missing from source, (c) line is structural and has no tokens. Proposal: empty token stacks in pending-state tablets get a subtle diagonal-stripe overlay pattern + "token readings pending" label in the translation ribbon. This sets accurate expectations and prevents scholars from assuming the data is wrong.
Marcus Chen (hobbyist) · All new users
╔══════════════════╗ ║ ░░░ processing ░░░ ║ ╚══════════════════╝ ● token readings pending (pipeline step 10)
08 Medium Priority
Rubric protocol marker flags — !qt, !cm, !bs
ATF uses rubric markers (!qt = quotation, !cm = commentary line, !bs = base text, !zz = uncertain genre) to annotate scholarly context inline. These are absent from the current Token Flags design. A purple CM / QT / BS chip in the flags row would let scholars immediately identify quotation and commentary layers — critical for literary and canonical texts where these distinctions determine textual hierarchy.
Dr. Al-Rashid · Canonical text specialists
┌ flags ┐ CM — commentary line (!cm) QT — quotation (!qt) BS — base text (!bs)
09 Medium Priority
Per-line attribution tooltip on row-number hover
Glintstone tracks which scholar contributed each line's ATF transcription via annotation_runs. This data is never surfaced in the current UI. A hover tooltip on the row number showing "ATF: Englund 2001 · Trans: Heimpel 1990" would support granular scholarly crediting — critical for a platform that aims to be academically citable. Especially important for P034009 (3 distinct contributors per field) and P336287 (rich ORACC credits).
Dr. Al-Rashid (attribution) · Aisha (citation)
10 Medium Priority
Line selection + export for dissertation workflow
Graduate scholars (Aisha persona) need to extract specific lines for dissertation preparation. Currently there is no selection mechanism. Proposal: on row hover, show a checkbox in the row-number gutter; selected rows highlight; a floating toolbar appears offering "Copy as ATF", "Copy as transliteration", and "Export citation (BibTeX/Chicago)". This turns the viewer into a research tool, not just a reading interface.
Aisha Okonkwo (PhD candidate) · Educators
[✓] 3 tu₇ gu₂ gal / broad bean soup [✓] 4 tu₇ gu₂ tur / lentil soup ───────────────────────────── Copy ATF | Copy trans | Cite
11 Lower Priority
Parallel exemplar compare mode
When a tablet is an exemplar of a composite (Q-number), scholars frequently want to compare it against another exemplar of the same text. A "Compare with exemplar" button in the header (enabled when artifact_composites linkage exists) could open a split-pane view showing two tablets side by side. This is how most Assyriologists work with lexical texts — comparing witness manuscripts is the primary scholarly task. Currently requires switching between tabs or browser windows.
Dr. Al-Rashid · Aisha (text tradition analysis)
12 Lower Priority
Damage gradient — visual opacity spectrum for text certainty
Currently damage is binary: either the # flag appears or it doesn't. ATF actually encodes a spectrum: sign (certain) → sign# (damaged but read) → sign? (uncertain) → [sign] (broken, restored by editor) → [x] (broken, unreadable) → [...] (full lacuna). Mapping these states to a visual opacity or color ramp would let scholars immediately perceive text confidence. Full lacunae ([...]) at ~25% opacity; restored readings slightly dimmed; certain readings at full brightness.
All scholars assessing textual reliability
tu₇ — certain (100%) tu₇# — damaged (~75%) tu₇? — uncertain (~55%) [tu₇] — restored (~35%) [...] — lacuna (~20%)

UX Explore — Interlinear vs. Ribbon Mode

The Ribbon mode (current design) is optimised for reading — each token is a self-contained column, ideal for scanning a text horizontally. The Interlinear mode rotates this 90°: each linguistic layer (ATF / NORM / LEMMA / GLOSS) becomes its own horizontal band, aligned by token position. This is the standard format in published grammars, the CAD, and philological commentary — the format scholars already read and write in. Toggle below to switch between them on the same data.

Show analysis layers
Interlinear In interlinear mode, scanning vertically down the LEMMA band gives an instant overview of all verbal forms in the passage — critical for checking aspect, tense, and person marking across a whole line or section. In ribbon mode, those same lemmata are scattered across the horizontal token columns and cannot be read as a sequence.
Mid-line switch Row 5 has a mid-line Sumerian → Akkadian switch. In interlinear mode the sux / akk chips appear in the ATF row exactly where the switch occurs — readable without a separate separator band. In ribbon mode the same information requires the language-border treatment on each token column.

When to use each mode

Ribbon mode

Better for reading and navigating a text

  • Horizontal flow: matches how a tablet is physically read — left to right across columns
  • Dense layout: more lines visible at once; good for long texts (e.g., 265-line lexical lists)
  • Context switching: a scholar jumping between sections benefits from maximum line density
  • Casual reading: Marcus / Sophie personas — come for the translation, not the morphology
  • Mobile-friendly: columns stack better on narrower screens if vertically condensed

Interlinear mode

Better for philological analysis and comparison

  • Layer scanning: read all lemmata, all normalisations, or all glosses as a single sequence — matches how commentary and grammars are typeset
  • Comparison against editions: Dr. Al-Rashid compares Glintstone's LEMMA row against CAD citations — same visual alignment as printed interlinear editions
  • Morphological teaching: David Yamamoto (educator) uses this format in classroom glossing exercises
  • Verbal paradigm checking: spotting tense/aspect/person patterns across a passage requires seeing all V.PRES in sequence
  • Dissertation quoting: Aisha's thesis excerpts use Leipzig-style interlinear — she can screenshot directly
Impl note The column widths in interlinear mode need to stay in sync across layers — each token position's minimum width must accommodate its widest item across all four rows (ATF, NORM, LEMMA, GLOSS). This is straightforward with CSS grid grid-template-columns where the token grid is the parent and each layer is a display: contents child, letting native grid alignment do the work with no JavaScript measurement loop.

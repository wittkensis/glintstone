User Personas

These personas represent the spectrum of end-users for the Glintstone platform. They are designed for simulated user validation, user testing, and product decision-making.

Primary Personas

Core users the platform must serve well from launch.

Primary

Marcus Chen

"The Weekend Warrior"

Tech professional with 15 minutes to spare. Wants meaningful contribution without expertise. Represents the largest potential user base.

Primary

Dr. Fatima Al-Rashid

"The Domain Expert"

Senior Assyriologist and expert reviewer. Gatekeeper of quality. If experts don't trust the platform, hobbyist contributions mean nothing.

Primary

Aisha Okonkwo

Primary

Usage Guide

For Simulated User Testing

  • Stay in character: Maintain the persona's knowledge level, time constraints, and motivations
  • Apply realistic friction: Personas with less time should abandon quickly if confused
  • Track appropriate metrics: Use the success metrics defined for each persona
  • Test edge cases: What happens when they're interrupted? When they disagree?

For Product Decisions

  • Weight by segment size: Marcus represents the largest potential user base; Dr. Al-Rashid the smallest but most critical
  • Trust is foundational: If experts don't trust the platform, hobbyist contributions mean nothing
  • Design for extremes: If Marcus and Dr. Al-Rashid can both use a feature, Aisha probably can too

For Agent Prompts

  • Each persona HTML file contains structured JSON in a hidden <script> block
  • Key data is in semantic HTML with data-* attributes for parsing
  • Test scenarios include unique IDs for tracking
